# Inbox

Use this file for rough capture. It is intentionally low-friction: paste notes here before they disappear.

The goal is not to make every note clean immediately. The goal is to avoid losing useful thoughts, then periodically refine them.

## Capture Rules

- Add the date when possible.
- Keep rough notes short, but do not over-polish them.
- Mark uncertainty explicitly.
- Add a `Next review` hint if you already know what should happen.

## Quick Capture Template

```text
## YYYY-MM-DD - Short title

Type: concept | project idea | question | debugging note | resource | decision | random
Status: raw

Note:
-

Why it might matter:
-

Next review:
- refine into knowledge base
- turn into quick project
- connect to existing project
- close as not useful
```
