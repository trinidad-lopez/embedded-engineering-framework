---

## 1. Purpose

This domain exists to build practical embedded systems engineering capability.

The goal is to improve from basic embedded programming toward reliable firmware design, debugging, architecture, communication protocols, real-time behavior, and portfolio-quality embedded projects.

This domain is the original foundation of the framework and remains important for career growth, work performance, and long-term technical depth.

---

## 2. Scope

This domain includes:

- C/C++ for embedded systems
- microcontroller fundamentals
- bare-metal programming
- peripheral drivers
- communication protocols
- sensors and actuators
- debugging techniques
- RTOS fundamentals
- firmware architecture
- embedded project design
- hardware/software interaction
- testing and validation
- documentation for embedded projects
- portfolio-oriented embedded work

This domain does not include yet:

- full custom silicon or physical design
- advanced analog IC design
- large-scale distributed systems
