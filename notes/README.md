# Notes

This folder is the repo-owned capture and refinement area.

Use `docs/inbox.md` for the fastest single-file capture flow. Use this folder when a note is large enough to deserve its own file or when the workbench should inspect it as a separate artifact.

## Folders

- `inbox/`: raw notes, pasted fragments, questions, and unsorted ideas.
- `concepts/`: reviewed concept notes that can feed the knowledge base.
- `project-ideas/`: rough or scoped project ideas before they become active projects.
- `workbench/`: notes created during workbench sessions or integration tests.

## Naming

Prefer dated, readable names:

```text
YYYY-MM-DD-short-title.md
```

Example:

```text
2026-06-15-uart-framing-confusion.md
```

## Review Rule

Every reviewed note should end in one of these outcomes:

- linked to a concept
- linked to a project
- moved into a debugging case
- converted into a session log
- closed with no action
